# Публичные экраны — Документ для передачи

## Обзор

Ты отвечаешь за **4 публичных дисплея** для системы Beachside Racetrack. Это большие экраны (40-75 дюймов), расположенные в зонах для зрителей и гонщиков. Они **только читают данные** — никогда не отправляют команды.

**Аутентификация не нужна** для публичных экранов.

---

## Твои 4 экрана

### 1. `/leader-board` (Зрители)

Таблица лидеров в реальном времени:

- Список машин/гонщиков, отсортированных по лучшему кругу
- Текущее количество кругов для каждой машины
- Лучшее время круга для каждой машины
- Таймер обратного отсчёта гонки (оставшееся время)
- Индикатор текущего флага/режима (safe/hazard/danger/finish)
- Показывать результаты предыдущей гонки пока не начнётся следующая

### 2. `/next-race` (Гонщики)

Показывает кто гоняет следующим:

- Список гонщиков следующей сессии с назначенными номерами машин
- Когда судья нажимает "Start Race" → переключить на показ СЛЕДУЮЩЕЙ сессии
- Когда судья нажимает "End Session" → показать текущую сессию с сообщением "Proceed to the paddock"

### 3. `/race-countdown` (Гонщики)

- Большой полноэкранный таймер обратного отсчёта
- 10 минут в продакшне, 1 минута в dev-режиме

### 4. `/race-flags` (Гонщики — заменяет живых флагоносцев)

Полноэкранный флаг, никакой другой информации:

| Режим   | Отображение            |
|---------|------------------------|
| Safe    | Сплошной зелёный       |
| Hazard  | Сплошной жёлтый        |
| Danger  | Сплошной красный       |
| Finish  | Шахматный чёрно-белый  |

### На всех 4 экранах должна быть **кнопка полноэкранного режима**.

## Как подключиться

Все экраны подключаются через **Socket.IO** к бэкенд-серверу.

```typescript
import { io } from 'socket.io-client';

// Подключение без auth (публичным экранам он не нужен)
const socket = io('http://localhost:3001');

// Слушаем обновления состояния гонки
socket.on('race-control:state', (data) => {
  console.log(data);
  // Обновляй UI здесь
});
```

Сервер отправляет `race-control:state` сразу при подключении и при каждом изменении состояния.

## Формат данных — `race-control:state`

Это **единственное событие**, которое нужно слушать. Вот TypeScript интерфейс:

```typescript
interface RaceState {
  sessionId: string | null;       // null когда нет активной гонки
  sessionName: string | null;     // например "Race 1"
  mode: RaceMode;                 // текущий режим гонки
  startedAt: string | null;       // ISO дата старта гонки
  endedAt: string | null;         // ISO дата завершения сессии
  participants: Participant[];    // гонщики в текущей гонке
  bestLap: BestLap | null;        // лучший круг за гонку
}

type RaceMode = 'idle' | 'safe' | 'hazard' | 'danger' | 'finish' | 'ended';

interface Participant {
  name: string;    // имя гонщика
  laps: number;    // сколько кругов проехал
}

interface BestLap {
  time: string;        // время круга строкой
  racerName: string;   // кто поставил лучший круг
}
```

## Жизненный цикл режимов гонки

```
idle → safe → hazard/danger (можно переключаться между ними) → finish → ended → idle
```

- **idle**: Гонка не идёт. Показывай предыдущие результаты или "ожидание".
- **safe**: Гонка активна, зелёный флаг.
- **hazard**: Жёлтый флаг, ехать медленно.
- **danger**: Красный флаг, остановиться.
- **finish**: Шахматный флаг, машины возвращаются в пит-лейн. Подсчёт кругов ещё работает.
- **ended**: Сессия полностью завершена. Следующая сессия в очереди.

## Расчёт таймера

Бэкенд отправляет `startedAt` (ISO дата). Чтобы посчитать оставшееся время:

```typescript
const RACE_DURATION = 600; // 600 секунд = 10 мин (продакшн)
// const RACE_DURATION = 60; // 60 секунд = 1 мин (dev)

function getRemainingSeconds(startedAt: string | null): number {
  if (!startedAt) return RACE_DURATION;
  const elapsed = Math.floor((Date.now() - new Date(startedAt).getTime()) / 1000);
  return Math.max(0, RACE_DURATION - elapsed);
}
```

## Стек

Используй любую фронтенд-технологию (React, vanilla JS и т.д.). Требования:

- Socket.IO client для real-time данных
- Роуты: `/leader-board`, `/next-race`, `/race-countdown`, `/race-flags`
- Кнопка fullscreen на каждом экране
- Никакого polling — только real-time обновления

---

## Быстрый старт

1. `cd apps/web`
2. Настрой свой фронтенд-проект
3. `npm install socket.io-client`
4. Подключись к `http://localhost:3001`
5. Слушай `race-control:state`
6. Делай свои 4 экрана
